# C2 Educational Prototype

Минимальный прототип Command & Control фреймворка в учебных целях. Сервер на Python (Flask) с веб-интерфейсом, клиент (beacon) на C# — компилируется в один `.exe` файл.

## Как это работает

Архитектура построена на стандартном HTTP polling паттерне, который используется в большинстве реальных C2 фреймворков:

```
┌──────────────┐       HTTP (каждые 3 сек)        ┌──────────────┐
│    Beacon     │ ─────────────────────────────► │    Server     │
│  (beacon.exe) │ ◄───────────────────────────── │   (Flask)     │
│               │   задача / нет задачи           │               │
│               │ ─────────────────────────────► │               │
│               │   результат выполнения          │         :5000 │
└──────────────┘                                  └───────┬───────┘
                                                          │
                                                     Web UI│
                                                          │
                                                  ┌───────▼───────┐
                                                  │   Оператор     │
                                                  │  (браузер)     │
                                                  └───────────────┘
```

**Цикл работы пошагово:**

1. **Beacon** запускается на целевой машине и регистрируется на сервере (отправляет hostname, username, ОС).
2. **Beacon** входит в бесконечный цикл — каждые 3 секунды делает HTTP запрос к серверу: "Есть задача для меня?"
3. **Оператор** открывает веб-интерфейс сервера в браузере, видит подключённых клиентов и отправляет команды.
4. **Сервер** ставит команду в очередь. При следующем опросе beacon забирает задачу.
5. **Beacon** выполняет задачу локально и отправляет результат обратно на сервер.
6. **Сервер** сохраняет результат. Веб-интерфейс подхватывает его и показывает оператору.

## Структура проекта

```
dedoLom2/
├── server.py              # Сервер + веб-интерфейс (сторона оператора)
├── requirements.txt       # Python зависимости для сервера
├── client/                # C# проект клиента (beacon)
│   ├── Program.cs         # Исходный код beacon
│   ├── client.csproj      # Файл проекта .NET

```

## Требования

**Для сервера (машина оператора):**
- Python 3.8+
- pip

**Для сборки beacon (нужно только для компиляции, не для запуска):**
- .NET 8 SDK (`winget install Microsoft.DotNet.SDK.8`)

**Для запуска beacon (целевая машина):**
- Только Windows x64. Никаких зависимостей — всё встроено в .exe

## Установка и запуск

### Шаг 1. Установить зависимости сервера

```bash
pip install -r requirements.txt
```

Это установит Flask (веб-фреймворк) и requests.

### Шаг 2. Запустить сервер

```bash
python server.py
```

В терминале появится:
```
==================================================
  C2 Educational Prototype - Server
  http://localhost:5000
==================================================
```

Открой `http://localhost:5000` в браузере — это панель управления.

### Шаг 3. Запустить beacon

**Вариант A — готовый .exe (рекомендуется):**

Если уже скомпилирован, просто запусти:
```
client\bin\Release\net8.0\win-x64\publish\beacon.exe
```

**Вариант B — скомпилировать и запустить:**

```bash
cd client
dotnet publish -c Release
bin\Release\net8.0\win-x64\publish\beacon.exe
```

### Шаг 4. Использовать веб-интерфейс

После запуска beacon в течение нескольких секунд он появится в боковой панели веб-интерфейса.

## Использование веб-интерфейса

### Подключённые клиенты (левая панель)

- Каждый beacon отображается как карточка с hostname, username и ID
- **Зелёная точка** — beacon активен (отвечал в последние 15 секунд)
- **Серая точка** — beacon не отвечает (потерял связь или завершён)
- Кликни на карточку, чтобы выбрать клиента для управления

### Кнопки управления (верхняя панель)

| Кнопка | Действие |
|--------|----------|
| **List Files** | Показать файлы и папки в текущей директории beacon |
| **Go Up** | Перейти в родительскую директорию (`cd ..`) |
| **PWD** | Показать текущую рабочую директорию beacon |
| **CD to path...** | Перейти по указанному пути (абсолютному или относительному) |
| **Download file...** | Скачать файл по указанному пути |

### Навигация по файлам

- **Клик по папке** — входит в неё (выполняет `cd` + `ls`)
- **Клик по файлу** — скачивает файл через сервер на твой компьютер
- Размеры файлов отображаются справа
- Иконки: папки и файлы визуально различаются

## Доступные команды

|---------|----------|--------|
| `ls` | Список файлов и папок | — |

## Настройка адреса сервера

По умолчанию beacon подключается к `http://localhost:5000` (тот же компьютер).

Чтобы подключаться к серверу на другой машине в сети, отредактируй строку в `client/Program.cs`:

```csharp
const string ServerUrl = "http://192.168.1.100:5000";  // IP сервера
```

После изменения пересобери:

```bash
cd client
dotnet publish -c Release
```

Новый `beacon.exe` будет в `bin/Release/net8.0/win-x64/publish/`.

## Пересборка beacon

Если изменил исходный код — пересобери:

```bash
cd client
dotnet publish -c Release
```

Параметры сборки (заданы в `client.csproj`):
- **PublishSingleFile** — всё в одном .exe
- **SelfContained** — .NET runtime встроен, не нужен на целевой машине
- **PublishTrimmed** — неиспользуемый код удалён для уменьшения размера
- **RuntimeIdentifier: win-x64** — собирается под Windows 64-bit

## Ключевые концепции C2

### HTTP Polling
Beacon не держит постоянное соединение с сервером. Вместо этого он периодически делает обычные HTTP запросы. Это стандартный подход в C2, потому что исходящий HTTP трафик выглядит как обычный веб-сёрфинг и не привлекает внимания сетевого мониторинга.

### Task Queue (очередь задач)
Сервер хранит одну ожидающую задачу на каждого клиента. Когда beacon делает опрос — забирает задачу, и очередь очищается. Это простейший вариант; реальные C2 поддерживают очередь из нескольких задач.

### Асинхронные результаты
Веб-интерфейс отправляет команду и затем опрашивает отдельный endpoint, ожидая появления результата. Это развязывает оператора и тайминг опроса beacon — оператор не ждёт синхронно.

### Передача файлов через Base64
Скачиваемые файлы кодируются в base64 и передаются внутри JSON ответа. Это стандартный приём для передачи бинарных данных через текстовые протоколы (HTTP/JSON).

### Self-Contained Executable
Beacon собирается как self-contained — .NET runtime встроен в .exe. Целевой машине не нужно ничего устанавливать. Это важно для C2, потому что beacon должен работать на любой Windows машине без подготовки.
